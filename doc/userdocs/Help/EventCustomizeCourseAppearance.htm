Event / Customize Appearance

The Customize Appearance command allows changing the appearance of the drawn course. Changes affect the live display, printed courses, and courses exported to OCAD/OOM.

 

The following items allow you to customize the appearance of the course. The "Preview" section shows a preview of what those changes will look like.

 

Use IOF standard sizes: If this is checked, Purple Pen uses the IOF standard sizes for control circles, legs, control numbers, and other items on the course. To change those sizes, uncheck this option.

Control circle diameter: Sets the outside diameter of control circles to the given size in mm. This also affects the size of the start triangle, finish circle, crossing points, refreshment marking, and first aid markings, which will be scaled in the same ratio. The IOF standard is 6.0 mm in ISOM2000, and 5.35mm in ISOM2017. (Note that ISOM2017 measures the circle diameter from the center of the line, while Purple Pen uses the outside diameter, so 5.35mm outside diameter is equivalent to 5.0mm diameter from the line centers.)

Line width: Sets the width of lines for legs, control circles, start triangle, and the finish circle. The IOF standard is 0.35 mm.

Center dot diameter: If set to something higher than 0.00m (the IOF standard), places a dot of the indicated diameter in the center of the control circle. Although not typically used, it is useful for certain types of maps.

Control number height: Sets the height of digits used for control numbers. The IOF standard is 4.0 mm.

Control number style: Determines the style of the control number. There are four possible styles:

  • Arial — Unbolded in the Arial font. This is specified in the IOF standard defined in ISOM 2017, so use this option if strict adherance to the ISOF 2017 standard is important to you.
  • Arial Bold — Boldface in the Arial font.
  • Roboto — Unbolded in the Roboto font. This is the default. Although not the IOF standard, the Roboto font is more modern looking, better distinguishes digits such as 8 and 9, 5 and 6, and matches the font used in the control descriptions.
  • Roboto Bold — Boldface in the Roboto font.

White outline around numbers: If greater than zero, a white outline of the given width is placed around numbers to make them more visible against colored or complex backgrounds. A value between 0.1mm and 0.4mm usually works best.

Automatic leg gap size: If greater than zero, controls how much of a gap to automatically place when one leg crosses another leg. Set this to zero to disable placing automatic gaps in legs.

Scale item sizes: Determines how the item sizes (circle diameter, line width, etc.) are modified based on the selected print scale of the course. There are three possible options:

  • Do not scale. The item sizes will always print at the given size on the paper, no matter the print scale. This option was how Purple Pen always worked prior to version 3.1.
  • Relative to map scale. The item sizes will scale along with the map if the print scale is different than the map scale. For example, if the map scale is 1:10,000, and the print scale is 1:7,500, then circles, line width, etc. will be 1.333x of the given size. If the print scale is the same as the scale of the underlying map, then items will print at the given size.
  • Relative to 1:15,000. The item sizes will scale based on the printing scale relative to a 1:15,000 base resolution, regardless of the scale of the underlying map. For example, if the map is 1:10,000, and also printed at 1:10,000, then circles, line width, etc. will be at 1.5x of the given size. This is consistent with the ISOM2017 mapping specification.

Blend purple with underlying map colors: There are three possible settings for this option:

  • None: The course lines and control number overlay the map without blending.
  • Blend: The course lines and control numbers are blended with the underlying map. This allows features in the map (especially black features) to show through the purple. To prevent blending of control numbers only, set the "white outline around numbers" setting to a value greater than zero.
  • Layer: The course lines and control numbers are placed in a map layer intermediate to other features on the map. This allows features in the map (especially some line and point features) to print above the purple. To be precise, purple objects that are designated as "lower purple" are placed in a map layer directly above the one selected in "Above map layer". Purple pen attempts to pick a good map layer automatically, but depending on the color table in your map, you may need to adjust the map layer for best results. This option generally produces the best results. It is not available if the underlying map is a bitmap or PDF.

Use purple color from map: If this is checked, the color used for the course will be the same shade of Purple as used in the underlying OCAD or OOM map. If an image is used instead of an OCAD map, or the Purple color cannot be determined, a default purple color is used. Uncheck this to specify the purple color directly.

Cyan, Magenta, Yellow, Black: Specifies the percentages of each pigment used to create the purple color, from 0 to 100.

Control Descriptions, Color: Specifies whether control descriptions on the map are displayed in black or purple. The default is black.

Use overprint effect for colors marked "overprint": Specifies whether a color "blending" or "overprinting" effect is applied for OCAD or OOM colors that have the "overprint" check mark applied to them. (In Open Orienteering Mapper, the option is called "knockout" and behaves in the reverse way). Since OCAD does not display this result on screen, and its effect on printing is inconsistent, enabling this option may produce bad results with some maps. Purple Pen uses this option for screen display, printing, and PDF export. Turn this option on only if you are sure that the color table of your map is correctly created. Note that the IOF no longer recommends overprinting or blending colors when printing.

 

Note: the sizes of control circles and other feature are calculated based on the scale the map will be printed at. If the circles appear too large or too small, it is likely that the map scale or print scale is set incorrectly, or (if an image file is used for the map) the dots per inch of the image is set incorrectly. This type of error will likely result in the length of the courses being calculated incorrectly as well, so it is highly recommended to fix the scale or dots per inch instead of adjusting the circle sizes.
